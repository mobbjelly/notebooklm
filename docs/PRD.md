# 产品需求文档（PRD）

**产品名称：** KnowBase（暂定）
**文档版本：** v0.1
**日期：** 2026-04-24
**阶段：** MVP

---

## 一、产品概述

### 1.1 背景与目标

Google NotebookLM 验证了「将 AI 与个人知识库深度结合」的方向，但其在中文语义理解、国内数据合规、国产模型生态接入等方面存在明显短板。

本产品面向**中文个人用户**，提供以文档为核心的 AI 知识整理工具，帮助用户**高效沉淀、消化、关联个人知识资料**。

### 1.2 产品定位

> 「中文用户的 AI 知识管理助手：上传文档，AI 帮你读懂、总结、关联。」

### 1.3 MVP 验证目标

- 验证核心功能（文档上传 + AI 问答 + 摘要生成 + 多文档关联）是否满足个人知识整理需求
- 积累种子用户，收集真实反馈
- 商业模式暂不考虑，先跑通产品价值

## 二、目标用户

### 2.1 核心用户画像

| 维度 | 描述                                             |
| ---- | ------------------------------------------------ |
| 人群 | 知识工作者、学生、研究人员、内容创作者           |
| 场景 | 个人整理知识资料：读书笔记、行业资讯、学习材料等 |
| 痛点 | 资料分散难管理、读完遗忘、无法快速检索与关联     |
| 期望 | 上传资料后 AI 自动帮我整理、总结、回答问题       |

### 2.2 用户使用场景（核心）

1. **碎片知识整理**：将散落在各处的 PDF、网页、文章统一上传，AI 帮助归纳主题
2. **深度阅读辅助**：上传长文/书籍，通过对话快速找到关键信息
3. **跨资料关联发现**：多份资料一起分析，AI 识别共同主题、矛盾点或知识图谱

## 三、功能需求

### 3.1 功能全景（MVP 范围）

```
KnowBase MVP
├── 笔记本（Notebook）管理
│   ├── 创建 / 删除 / 重命名笔记本
│   └── 笔记本列表与搜索
├── 文档管理
│   ├── 支持格式：PDF、Word、TXT、Markdown、网页 URL
│   ├── 上传 / 删除文档
│   └── 文档解析状态展示
├── AI 核心能力
    ├── 文档 AI 问答（对话式）
    ├── 自动摘要生成
    ├── 自动笔记生成（结构化要点）
    └── 多文档关联分析
```

### 3.2 功能详细说明

#### F01 — 笔记本管理

| 需求项     | 描述                                                                   |
| ---------- | ---------------------------------------------------------------------- |
| 创建笔记本 | 用户创建一个"知识容器"，可命名、添加描述                               |
| 文档归属   | 每个文档属于某一笔记本，支持同一文档加入多个笔记本（可选，MVP 可暂缓） |
| 笔记本列表 | 卡片式展示，含文档数量、最近更新时间                                   |

#### F02 — 文档上传与解析

| 需求项       | 描述                                         |
| ------------ | -------------------------------------------- |
| 支持格式     | PDF、DOCX、TXT、MD、网页 URL                 |
| 单文件大小   | MVP 阶段暂不限制，视服务器负载调整           |
| 解析状态     | 上传后展示「解析中 → 完成/失败」进度反馈     |
| 网页 URL     | 输入 URL 后自动抓取正文内容，过滤广告/导航栏 |
| 中文文档处理 | 针对中文 PDF 优化 OCR/解析，支持繁简体       |

#### F03 — AI 文档问答

| 需求项   | 描述                                                |
| -------- | --------------------------------------------------- |
| 对话界面 | 类似 ChatGPT 的多轮对话，问题基于当前笔记本内的文档 |
| 引用溯源 | AI 回答附带原文来源（文档名 + 段落）方便验证        |
| 问答范围 | 可选「当前文档」或「笔记本全部文档」                |
| 中文优化 | 理解中文专业术语、同义词、口语表达                  |

#### F04 — 自动摘要生成

| 需求项   | 描述                                               |
| -------- | -------------------------------------------------- |
| 触发方式 | 文档解析完成后自动生成，也可手动重新生成           |
| 摘要格式 | 结构化输出：核心观点（3-5条）+ 全文摘要（200字内） |
| 长文支持 | 超长文档（书籍级）分章节摘要                       |
| 可编辑   | 用户可在摘要上直接编辑、标注                       |

#### F05 — 自动笔记生成

| 需求项   | 描述                                                  |
| -------- | ----------------------------------------------------- |
| 笔记类型 | AI 自动提炼：关键概念、重要论点、数据引用、待深入问题 |
| 格式     | Markdown 格式，可导出                                 |
| 用户编辑 | 生成后支持用户修改、添加自己的注释                    |

#### F06 — 多文档关联分析

| 需求项   | 描述                                              |
| -------- | ------------------------------------------------- |
| 触发方式 | 笔记本中有 ≥2 个文档时，显示「关联分析」入口      |
| 分析内容 | 共同主题发现、观点异同对比、知识盲点提示          |
| 展示形式 | 结构化文字报告（MVP）；知识图谱可视化（后续版本） |

### 3.3 模型与技术要求

| 需求项     | 描述                                               |
| ---------- | -------------------------------------------------- |
| 支持模型   | 通义千问                                           |
| 数据存储   | 国内服务器部署，文档数据不传境外                   |
| RAG 架构   | 基于向量数据库的检索增强生成，确保回答基于文档内容 |
| 中文向量化 | 使用中文优化的 Embedding 模型（如 BGE、text2vec）  |

## 四、非功能需求

| 类别     | 要求                                                           |
| -------- | -------------------------------------------------------------- |
| 性能     | 单文档（≤50页 PDF）解析时间 < 30秒；AI 回答首 Token 延迟 < 3秒 |
| 隐私合规 | 用户文档数据隔离存储，不用于模型训练，符合《数据安全法》       |
| 可用性   | 界面语言全中文，操作路径 ≤ 3 步完成核心功能                    |
| 兼容性   | Web 端优先，响应式适配移动端（MVP 可只做 Web）                 |

## 五、用户体验关键路径

**核心 Happy Path（MVP 验证流程）：**

```
注册/登录
  → 创建笔记本
    → 上传 PDF 或粘贴 URL
      → 等待解析（进度提示）
        → 查看 AI 自动摘要 + 笔记
          → 对话提问
            → 查看引用来源
              → （多文档时）查看关联分析
```

## 六、范围边界

### MVP 包含

- 上述 F01~F06 全部功能
- Web 端产品
- 支持 PDF / DOCX / TXT / MD / URL

### MVP 不包含（后续版本）

- 音频/播客生成（类 NotebookLM Audio Overview）
- 团队协作与共享
- 知识图谱可视化
- 微信公众号/小红书内容接入
- 移动端 App
- 商业化与付费体系

## 七、风险与假设

| 风险              | 描述                            | 应对                                       |
| ----------------- | ------------------------------- | ------------------------------------------ |
| AI 幻觉           | 模型捏造文档中不存在的内容      | 强制要求引用溯源，仅基于文档回答           |
| 中文 PDF 解析质量 | 扫描件、复杂排版解析效果差      | 提前告知用户格式限制，后续引入 OCR 增强    |
| 成本控制          | 免费开放期 Token 消耗可能超预期 | 设置单用户每日问答次数软限制（如 50次/天） |
| 国产模型稳定性    | 各模型 API 可用性不一           | 支持多模型切换，设置 fallback 机制         |
